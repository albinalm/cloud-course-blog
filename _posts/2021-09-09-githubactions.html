What is GitHub Actions?

GitHub actions is a tool like Jenkins that is used to automate workflow in a project. This is done through a so called pipeline where you queue different events to be triggered depending on the configuration.

In this article I'll primarily talk about a CI(Continuous Integration) pipeline.

Continuous Integration Pipeline

Basically a CI Pipeline is a method or rather an approach to automate workflow with specific steps so you can deliver a new version of your software. Usually a CI Pipeline is combined with CD(Continuous delivery & Continuous deployment) which adds a few more steps, but I'll stick to talking about the part of the process.

Continuous Integration combines three vital steps; Build, Test & Merge. Through these steps everytime you update your application to your master branch the software will automatically be built, code tested and then merged. If any of these steps fails, the process will be stopped. So it is important that you program using TDD(Test-driven development) to make the most of this type of pipeline.
